# 🚀 Деплой сайта Соколовой Л.И. — пошаговая инструкция

Получишь ссылку вида `https://sokolova-finance.onrender.com`, по которой будет работать сайт + админ-панель. **Полностью бесплатно**, без банковской карты.

> ⏱ Займёт ~15 минут. Никаких терминалов — всё через браузер.

---

## Шаг 1. Загрузить код на GitHub

Если у тебя ещё нет аккаунта на GitHub — зарегистрируйся на https://github.com (бесплатно, нужна только почта).

### Создаём репозиторий

1. Зайди на https://github.com/new
2. Заполни:
   - **Repository name**: `sokolova-finance`
   - **Visibility**: ✅ **Public** (для бесплатного деплоя на Render это обязательно)
   - Остальное не трогай
3. Нажми зелёную кнопку **Create repository**

### Загружаем файлы

После создания репозитория увидишь страницу с инструкциями. Найди ссылку **«uploading an existing file»** (она в верхней части страницы, в синем блоке).

1. Распакуй у себя на компьютере архив `sokolova-db-mobile.zip`
2. Открой папку `sokolova-db` — внутри файлы и подпапки (`app`, `components`, `lib`, `package.json`, и т.д.)
3. **Перетащи всё содержимое** этой папки в окно браузера GitHub (выдели всё внутри `sokolova-db` мышью и тащи)
4. Внизу страницы напиши Commit message: `initial commit`
5. Нажми зелёную кнопку **Commit changes**

⚠️ **Не загружай**: папку `node_modules` (если она есть), папку `.next`, папку `data`, файл `.env.local`. Они не нужны и слишком большие.

После загрузки в репозитории должна быть видна папка `app`, `components`, `lib`, файлы `Dockerfile`, `render.yaml`, `package.json` и т.д.

---

## Шаг 2. Регистрация на Render

1. Зайди на https://render.com
2. Нажми **Get Started for Free**
3. Выбери **GitHub** для входа (это упростит подключение репозитория)
4. Разреши Render доступ к твоему GitHub
5. Карту привязывать **НЕ нужно** — для бесплатного плана не требуется

---

## Шаг 3. Создаём веб-сервис

После входа в Render:

1. На главной странице нажми синюю кнопку **+ Add new** (вверху справа) → **Web Service**
2. На странице **Source Code** выбери **Public Git repository**
3. Найди в списке свой репозиторий `sokolova-finance` и нажми **Connect**
   - Если репозитория нет в списке — нажми **Configure GitHub App** и дай Render доступ к нему

### Настройки сервиса

Заполни форму:

| Поле | Значение |
|---|---|
| **Name** | `sokolova-finance` (или любое — это будет в URL) |
| **Region** | `Frankfurt (EU Central)` — ближе всего к России |
| **Branch** | `main` |
| **Runtime** | Должно автоматически определиться как **Docker** |
| **Instance Type** | ✅ **Free** |

### Переменные окружения

Прокрути ниже до раздела **Environment Variables**. Нажми **Add Environment Variable** и добавь:

| Key | Value |
|---|---|
| `ADMIN_SECRET` | придумай пароль для админки, например `my-super-pass-2026` |

⚠️ **Запомни этот пароль** — он понадобится для входа в `/admin`.

### Запуск деплоя

Нажми внизу страницы **Deploy Web Service**.

Render начнёт сборку — это занимает 5–10 минут на первый раз. В логах увидишь, как:
1. Скачивается код
2. Собирается Docker-образ (`docker build`)
3. Устанавливаются npm-пакеты
4. Собирается Next.js приложение (`npm run build`)
5. Запускается сервер

Когда увидишь в логах строку **`Your service is live 🎉`** и сверху появится зелёная плашка **Live** — готово.

---

## Шаг 4. Открываем сайт

Ссылка на сайт будет вверху страницы сервиса в Render, что-то вида:

```
https://sokolova-finance.onrender.com
```

(или `sokolova-finance-xxxx.onrender.com` если имя уже занято)

**Открой эту ссылку** — увидишь главную страницу сайта.

### Проверка админки

Зайди на адрес `https://sokolova-finance.onrender.com/admin`. Появится форма ввода пароля — введи тот `ADMIN_SECRET`, который сохранил в переменных окружения.

### Тестирование

1. На главной странице прокрути вниз до формы «Записаться» и заполни её
2. Нажми **Отправить заявку**
3. Зайди в `/admin` — увидишь свою заявку
4. Поменяй статус, удали — всё работает

---

## ⚠️ Особенности бесплатного плана Render

1. **Сайт «засыпает» после 15 минут без активности.** При первом обращении после сна загрузка занимает 30–60 секунд (сервер просыпается). Дальше работает нормально.

2. **База данных сбрасывается при перезапуске.** На бесплатном плане нет постоянного диска — данные о заявках хранятся только в текущей сессии. Для **демонстрации на защите** это абсолютно нормально: заполнил форму → сразу показал в админке.

3. **Лимит — 750 часов в месяц.** Этого хватает с запасом на демонстрационный сайт.

### Если нужно постоянное хранение данных

Переведи сервис на платный план **Starter — $7/мес** (~620₽/мес). Тогда:
- Сайт не засыпает
- Можно добавить постоянный диск (Disk, $1/мес за 1 ГБ) — данные не теряются
- В файле `render.yaml` раскомментируй секцию `disk` и пересобери

## 🔁 Обновление сайта

Когда захочешь что-то поменять:

1. Меняешь файл прямо в GitHub (через интерфейс — кнопка карандаша на файле) или загружаешь обновлённую версию
2. Render автоматически замечает изменения и пересобирает сайт за 5–10 минут

---

## 🆘 Если что-то пошло не так

**Сайт не открывается, ошибка 502 / «Application failed to respond»**
→ Скорее всего сервер ещё засыпал. Подожди 60 секунд и обнови страницу.

**В логах Render ошибки сборки**
→ Чаще всего проблема в том, что не загрузился `package.json` или `Dockerfile`. Проверь в GitHub, что все файлы на месте.

**При входе в /admin пишет «Неверный пароль»**
→ Перепроверь `ADMIN_SECRET` в Render: Environment → Edit → должен совпадать с тем, что вводишь. После изменения переменной нужно нажать **Manual Deploy → Deploy latest commit** наверху страницы.

**В админке пусто после ребилда**
→ Это нормально для бесплатного плана: данные не персистентные. Просто заполни форму заново для демонстрации.
