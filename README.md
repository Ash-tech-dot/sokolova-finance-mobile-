# Соколова Любовь — Финансовый консультант

Next.js 14 + TypeScript + SQLite (better-sqlite3)

## Быстрый старт

```bash
# 1. Установить зависимости
npm install

# 2. Создать файл с переменными окружения
cp .env.local.example .env.local
# Открыть .env.local и поменять ADMIN_SECRET на свой пароль

# 3. Запустить в режиме разработки
npm run dev
```

Сайт откроется на http://localhost:3000

## База данных

- Тип: **SQLite** (файл `data/leads.db` создаётся автоматически)
- Пакет: `better-sqlite3`
- Файл БД **не нужно создавать вручную** — создаётся при первом запуске

## Админ-панель

Откройте http://localhost:3000/admin

Введите пароль из `ADMIN_SECRET` в `.env.local`

**Возможности:**
- Просмотр всех заявок
- Фильтрация по статусу (Новые / В работе / Завершены)
- Изменение статуса заявки
- Удаление заявок
- Клик по строке — детали заявки

## API

| Метод | URL | Описание |
|-------|-----|----------|
| POST | /api/leads | Создать заявку (публичный) |
| GET | /api/leads | Список заявок (требует x-admin-secret) |
| PATCH | /api/leads/:id | Обновить статус (требует x-admin-secret) |
| DELETE | /api/leads/:id | Удалить заявку (требует x-admin-secret) |

## Деплой на Vercel

> ⚠️ SQLite не работает на Vercel (serverless). Для продакшена используйте:
> - **Vercel Postgres** (простая замена, бесплатный тариф)
> - **PlanetScale** / **Neon** (PostgreSQL)
>
> Напишите — помогу переключить на Postgres за 10 минут.

Для локального сервера (VPS/DigitalOcean) SQLite работает отлично.

## Структура

```
├── app/
│   ├── api/leads/          # API: POST/GET заявок
│   │   └── [id]/           # API: PATCH/DELETE по id
│   ├── admin/              # Панель управления
│   └── page.tsx            # Главная страница
├── components/             # React компоненты
├── lib/
│   └── db.ts               # База данных (SQLite)
├── data/
│   └── leads.db            # База данных (создаётся автоматически)
└── .env.local              # Переменные окружения (создать вручную)
```
